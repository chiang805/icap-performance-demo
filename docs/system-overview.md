# 系統總覽

本系統展示企業策略、人力資源與績效管理的整合資料模型與工作流程。第一版以 GitHub Pages 靜態網站展示，資料來源預設為 mock。

## 核心設計

- 功能權限與資料範圍權限分離。
- 員工主檔與登入帳號分離。
- 職務、職稱、職等分離。
- 組織工作項目可繼承展開到職務說明書，並保留頻率、權重與產出物。
- 任務型橫向組織可串接專案、委員會、改善小組與跨部門任務。
- 工作日誌整合人力資源專案設計，從待辦快速帶入專案任務、OGSM、KPI 與部門工作項目。
- 正式 KPI 數值不得由工作日誌直接修改，須經確認流程後更新。
- 正式資料以 `status` 停用、作廢、版本保存，不直接永久刪除。
- 所有審核與關鍵異動寫入 audit log。

## OGSM 結構

- `O`：Objective 最終目的，回答「當我們成功時，我們看起來像什麼樣子？」
- `G`：Goal 具體目標，符合 SMART 原則。
- `S`：Strategy 策略，描述資源取捨與達成路徑。
- `M_D`：Measure Dashboard 衡量指標，對應進度、結果與異常狀態。
- `M_P`：Measure Plans 行動計畫，包含負責單位、協作單位、期限與產出物。

## Dashboard 戰情

Dashboard 不只顯示燈號，也提供異常總表與點入後的細項資訊，包含異常內容、負責單位、根因、期限與下一步。

## 資料服務抽象

前端透過 `DataService` 取得資料：

- `MockDataService`：展示用資料，預設啟用。
- `SupabaseDataService`：未來接 Supabase。
- 未來可新增 SQL Server API adapter，保留同一組方法介面。
