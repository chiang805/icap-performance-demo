# 使用者指南

## 個人首頁與待辦

首頁提供今日待辦、逾期工作、即將到期、OGSM 待更新、KPI 待填報、工作日誌狀態、待審核事項與每日檢核清單。

待辦來源包含：

- 專案任務：由任務負責人與期限帶入。
- OGSM Plans：由 `M_P` 行動計畫的負責單位、協作單位、期限與產出物帶入。
- KPI 更新週期：由 KPI 設定與報告期間判斷。
- 審核退回：由審核流程自動產生。

## 工作日誌

按下「建立工作日誌」或待辦清單的「帶入日誌」，會切換到工作日誌建立畫面，並預先帶入人力資源專案資料：

- 日期
- 員工
- 任務
- 部門工作項目
- OGSM
- KPI
- 專案
- 實際工作內容
- 成果
- 工時
- 進度
- 問題
- 所需支援
- 下一步

工作日誌不直接修改正式 KPI 數值，須經確認後更新。

## 組織圖與工作職掌定位

此頁展示正式組織圖、部門核心任務、員工定位、主管與指定審核人。部門工作項目會作為職務說明書展開的來源。

## 橫向組織工作職掌定位

此頁展示人力資源專案、改善小組、委員會與跨部門任務的主要負責人、成員、投入比例、關聯 OGSM 與工作職掌。

## OGSM

OGSM 頁面提供四種視角：

- 樹狀檢視：展示 O、G、S、M(D)、M(P) 的上下關係。
- 網狀對應：展示多重支援、協作、共背與依賴關係，並加上主旨說明。
- Plans 作業：展示行動計畫輸入與更新方式。
- 獨立檢視：可切換整體 O、整體 G、整體 S、整體 P、整體 D 與擁有者資料。

## Dashboard 戰情

Dashboard 提供燈號摘要、異常總表與異常細項。點選異常後可查看根因、負責單位、期限與下一步。

## 職務說明書

職務說明書繼承組織內的工作項目，除項目內容外，也顯示頻率、權重、產出物與 K/S/A 能力。
