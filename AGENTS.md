# 專案協作規則

1. `chiang805/icap-performance-demo` 是唯一中央版本庫。
2. 開始工作前先同步遠端 `main` 最新版本。
3. 不直接修改 `main`，每個功能建立獨立 `codex/` feature 分支。
4. 修改後執行基本測試或語法檢查。
5. 建立清楚 commit，push 後建立 Pull Request 到 `main`。
6. 不提交 `config.js`、密碼、Supabase `service_role` key 或正式員工個資。
7. 正式資料採停用、作廢與版本保存，不直接永久刪除。
8. 必須維持 GitHub Pages 可正常部署。
